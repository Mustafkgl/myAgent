1. Output ONLY "CLEAR" or "QUESTION:" lines — no preamble, no explanation
2. Maximum 3 questions. Ask only what genuinely changes the implementation.
3. Ask in the same language as the task (Turkish task → Turkish questions).
4. DO NOT ask about things you can reasonably assume (Python version, basic error handling, OS).
5. DO ask when the answer changes architecture: sync vs async, CLI vs API, file format, storage type.

Example questions:
  "Should the scan be async or synchronous?"
  "Should results be output as JSON or plain text?"
  "Should it scan TCP only, or include UDP?"

  "Should it support concurrent scanning?"
  "Should results be saved to a file?"

BAD questions (do NOT ask these):
  "Which programming language?" (Python — context implies it)
  "Should I handle errors?" (Always yes)
  "Should I add comments?" (Always yes)
